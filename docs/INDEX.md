## 🔍 File Naming Conventions

### Directory Names
```
NN-descriptive-name/
```
- Start with two-digit number (01-99) for ordering
- Use lowercase letters and hyphens
- Descriptive but concise

### File Names
```
descriptive-name.md
```
- All lowercase
- Hyphens between words (kebab-case)
- Clear, self-documenting names
- No spaces or underscores

### Examples
✅ Good:
- `03-architecture/technical-decisions.md`
- `04-design/design-system.md`
- `05-development/prompt-history.md`

❌ Bad:
- `Technical Decisions.md`
- `design_system_v2.md`
- `Dev/notes.md`

## 📝 How to Add Documentation

### Adding to existing section
1. Create `NN-section-name/your-new-file.md`
2. Add entry to section's README.md
3. Add link to relevant INDEX.md section
4. Commit with message: `Docs: Add [topic] to [section]`

### Creating new section
1. Create directory: `NN-new-section/`
2. Create `NN-new-section/README.md` with section overview
3. Add link to top-level INDEX.md
4. Add relevant files
5. Update section table
6. Commit: `Docs: Add [section name] documentation`

### Updating documentation
1. Edit the `.md` file directly
2. Update related INDEX or README links if structure changes
3. Commit: `Docs: Update [topic]`
