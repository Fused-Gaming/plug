## 2026-07-22 - fix: Resolve data loading error by using direct imports (#16)

- Commit: `e380eff04a`
- Author: SupItsJ

## Summary
Fixes the "Cannot destructure property 'locations' of 'undefined'" error
that prevented the application from loading location data.

## Root Cause
The `locationData` adapter was attempting to fetch from
`/plug/src/data/seed-data.json`, a path that doesn't exist in the GitHub
Pages static deployment. This caused silent fetch failures, leaving
locations data undefined and triggering destructuring errors in
dependent components (Filters, LocationList, etc).

## Solution
Changed `locationData.js` to import locations directly from
`locations.js` instead of using network fetch. This eliminates the
network dependency and ensures data is available in all deployment
environments (local, dev, and GitHub Pages).

### Changes
- **src/adapters/locationData.js**: Replaced fetch-based loading with
direct ES6 import
  - Removed: `fetch('/plug/src/data/seed-data.json')` network request
- Replaced with: Direct import of `locationsData` from
`../data/locations`
  - Maintains Promise-based interface via `Promise.resolve()`

## Testing
- Build succeeds without errors
- All existing 36 security and unit tests pass
- Application now loads with correct location data in browser

## Deployment
Ready for merge to main. This fix addresses the critical blocker
preventing the app from functioning at https://plug.vln.gg
