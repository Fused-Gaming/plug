Updated project planning to explicitly document that:

CONSTRAINT:
- GitHub Pages serves static files only (no React runtime on server)
- No backend, no server-side rendering, no Node.js APIs

OUR SOLUTION:
- Pre-build React to static HTML/CSS/JavaScript with Vite
- GitHub Actions runs: npm install → npm run build → deploy dist/
- React app runs entirely client-side in browser
- Result: Full interactive map on plug.vln.gg with zero backend

Added:
- PROJECT_PLAN.md - Complete technical decision log
- Updated ROADMAP.md with deployment constraint notes
- Updated DEPLOYMENT.md with GitHub Pages static build flow

Why this works:
✅ Perfect for MVP (no backend complexity)
✅ Automatic deployment via GitHub Actions
✅ Full React interactivity in browser
✅ Easy to add backend later if needed

Critical for team:
⚠️ Always verify npm run build works before pushing
⚠️ All data must be client-side (hardcoded or fetch from external API)
⚠️ Cannot use Node.js APIs in browser code
