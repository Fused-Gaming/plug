# Contributing to Next Chapter Project Demo

Thank you for your interest in contributing! This is a time-constrained bootcamp submission, so we're focused on essential features only.

## Development Workflow

1. **Create a feature branch** from `main`
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** with clear, atomic commits
   ```bash
   git commit -m "Add: brief description of change"
   ```

3. **Test your changes**
   ```bash
   npm test
   ```

4. **Push and create a pull request**
   ```bash
   git push origin feature/your-feature-name
   ```

## Commit Message Guidelines

- Use `Add:` for new features
- Use `Fix:` for bug fixes
- Use `Docs:` for documentation
- Use `Refactor:` for code improvements without changing functionality
- Keep messages concise and descriptive

## Code Standards

- Format code with [your chosen formatter]
- Add tests for new functionality
- Update documentation for public APIs
- No console.log() in production code

## Scope Constraints

⚠️ **CRITICAL:** This is a 7-day bootcamp submission. Only work on items in the active milestone. Do not add:
- Additional features beyond the MVP
- Complex abstractions
- Performance optimizations not required for demo
- Advanced error handling for edge cases

If you have ideas for improvements, create an issue after the submission deadline.

## Questions?

Create an issue with the `question` label, or reach out to the team directly.
